🔗 테이블 관계

User ↔ Order

1:N 관계
한 사용자는 여러 주문을 가질 수 있음
users.email ← "order".email

Product ↔ WishList

1:N 관계
한 제품은 여러 위시리스트에 담길 수 있음
products.product_id ← wish_list.product_id

User ↔ WishList

1:N 관계
한 사용자는 여러 위시리스트를 가질 수 있음
users.email ← wish_list.email

Order ↔ OrderItem

1:N 관계
한 주문은 여러 주문아이템을 가질 수 있음
"order".order_id ← order_item.order_id

Product ↔ OrderItem

1:N 관계
한 상품은 여러 주문아이템에서 참조됨
products.product_id ← order_item.product_id

구현 과정

4명이 백엔드를 분담하여 구현한 내용

🔐 User & Auth 구현

담당: 이찬수

User 클래스 메서드
  • register() - 회원가입
  • login() - 로그인 및 JWT 발급
  • getUserInfo() - 회원정보 조회
  • updateUser() - 회원정보 수정
  • deleteUser() - 회원 탈퇴
  • logout() - 로그아웃 처리
JWT 인증 시스템
  • JWT 토큰 생성 및 검증
  • Spring Security 설정
  • 인증 필터 구현
  • 쿠키 생성 (HttpOnly, Secure)
  • 쿠키 삭제 (로그아웃 시)
테이블: users

id, email, password, name, address, role, created_at

🛍️ Product 구현

담당: 임정민

Product 클래스 메서드
  • createProduct() - 상품 등록
  • getProducts() - 상품 목록 조회
  • getProductById() - 상품 상세 조회
  • updateProduct() - 상품 정보 수정
  • deleteProduct() - 상품 삭제
Product 엔티티 구조
  • productId (PK) - 상품 고유 ID
  • productName - 상품명
  • price - 가격
  • description - 상품 설명
  • orderCount - 주문 횟수
  • productImage - 상품 이미지 URL
  • stock - 재고 수량
  • createdAt - 생성일시
테이블: products

product_id, product_name, price, description, order_count, product_image, stock, created_at

📦 Order & OrderItem 구현

담당: 천영우

Order 클래스 메서드
  • createOrder() - 주문 생성
  • getOrders() - 주문 목록 조회
  • getOrderById() - 주문 상세 조회
  • updateOrderStatus() - 주문 상태 변경
  • cancelOrder() - 주문 취소
OrderItem 클래스 메서드
  • addOrderItem() - 주문 아이템 추가
  • getOrderItems() - 주문 아이템 조회
  • updateQuantity() - 수량 변경
  • calculateSubtotal() - 소계 계산
테이블

orders: id, user_id, total_price, status, order_date, delivery_address

order_items: id, order_id, product_id, quantity, price

테이블: wishlist

id, user_id, product_id, quantity, added_at

🔐 JWT 보안 구현

인증 흐름

  1. 로그인 시 서버에서 JWT 토큰 발급
  2. 클라이언트는 토큰을 쿠키에 자동 저장
  3. API 요청 시 쿠키의 토큰을 헤더에 포함
  4. 서버에서 토큰 검증 후 응답
  5. 로그아웃 시 쿠키 삭제 및 토큰 무효화

보안 설정

  • HttpOnly 쿠키 사용 - XSS 공격 방지
  • Secure 플래그 - HTTPS에서만 전송
  • 토큰 만료 시간 설정 - 24시간
  • Spring Security로 엔드포인트 보호

🚫

CORS 에러 - 브라우저 보안 정책

Before

Postman = 테스트 도구 (허용)
브라우저 = 사용자 환경 (차단)
"localhost:3000에서 8080 못부른다고?"

After

CORS = 브라우저가 다른 서버 차단
해결: 백엔드에서 허가 설정
.allowedOrigins("http://localhost:3000")

🔗

API 메서드명 불일치

Before

프론트: getProductsList()
백엔드: getProducts()
테이블명도 서로 다름

After

Swagger로 문서화 후 통일
명명 규칙 정하고 공유
제약사항 체크리스트 작성

💾

예상치 못한 DB 에러들

Before

Supabase prepared statement 에러
테이블 제약조건 오류
JPA 매핑 문제

After

prepareThreshold=0 설정 추가
@Column 어노테이션 정확히 작성
DB 제약조건 문서화

🌿

Git 협업의 어려움

각자 main에 직접 push
충돌 발생 시 당황
누가 뭘 했는지 모름

After

브랜치 전략 도입 (feature/기능명)
PR 리뷰 후 merge
commit 메시지 규칙 통일
