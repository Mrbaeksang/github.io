🔐 User & Auth 구현

담당: 이찬수

  • register() - 회원가입
  • login() - 로그인 및 JWT 발급
  • getUserInfo() - 회원정보 조회
  • updateUser() - 회원정보 수정
  • deleteUser() - 회원 탈퇴
  • logout() - 로그아웃 처리
JWT 인증 시스템
  • JWT 토큰 생성 및 검증
  • Spring Security 설정
  • 인증 필터 구현
  • 쿠키 생성 (HttpOnly, Secure)
  • 쿠키 삭제 (로그아웃 시)
테이블: users

id, email, password, name, address, role, created_at

🔐 JWT 보안 구현

인증 흐름

  1. 로그인 시 서버에서 JWT 토큰 발급
  2. 클라이언트는 토큰을 쿠키에 자동 저장
  3. API 요청 시 쿠키의 토큰을 헤더에 포함
  4. 서버에서 토큰 검증 후 응답
  5. 로그아웃 시 쿠키 삭제 및 토큰 무효화

보안 설정

  • HttpOnly 쿠키 사용 - XSS 공격 방지
  • Secure 플래그 - HTTPS에서만 전송
  • 토큰 만료 시간 설정 - 24시간
  • Spring Security로 엔드포인트 보호

🚫

CORS 에러 - 브라우저 보안 정책

Before

Postman = 테스트 도구 (허용)
브라우저 = 사용자 환경 (차단)
"localhost:3000에서 8080 못부른다고?"

After

CORS = 브라우저가 다른 서버 차단
해결: 백엔드에서 허가 설정
.allowedOrigins("http://localhost:3000")

🔗

API 메서드명 불일치

Before

프론트: getProductsList()
백엔드: getProducts()
테이블명도 서로 다름

After

Swagger로 문서화 후 통일
명명 규칙 정하고 공유
제약사항 체크리스트 작성

💾

예상치 못한 DB 에러들

Before

Supabase prepared statement 에러
테이블 제약조건 오류
JPA 매핑 문제

After

prepareThreshold=0 설정 추가
@Column 어노테이션 정확히 작성
DB 제약조건 문서화

🌿

Git 협업의 어려움

누가 뭘 했는지 모름

PR 리뷰 후 merge
commit 메시지 규칙 통일
